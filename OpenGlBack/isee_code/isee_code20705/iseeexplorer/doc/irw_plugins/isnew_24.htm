SAVESTR 结构说明
 
 
SAVESTR 结构将用于主程序向插件的保存接口函数传递用户对新增下拉列表的选择数据。这个 结构非常简单，只有一个成员变量，而且它应用的范围也非常小，只会出现在插件的保存接 口函数的参数表中（最后一个入口参数项）。要理解本结构在插件代码中所起的作用，请参 看 IRW_SAVE_DESC 结构说明一节中对插件保存机制的说明。
 
变 量 名 含 义
int
para_value[ISD_ITEM_NUM]; 		用户对插件所有新增下拉列表的选择值。每一个插件都可以让主程序在保存对话框中新增一个 或数个保存参数下拉列表，用户在作出选择后，主程序会将每一个列表的选择值（即列表的当前索引值， 以0为基）存入本变量的对应元素中，也就是将第一个新增下拉列表的当前索引值存入本变量 的0下标元素中，第二个新增下拉列表的当前索引值存入本变量的1下标元素中，依此类推。插件 的保存接口函数将根据本变量中各元素的值到插件信息表的 irwp_save 变量中查找对应的实际 保存参数，然后再将这些实际参数应用到保存操作中。
变 量 名 含 义


当前在本结构中只有一个成员变量，不过在未来根据形式需要，可能会增加一些其它的成员变量。
